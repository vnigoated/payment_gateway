Invoice API — Developer Reference

Invoice API lets you programmatically generate GST-compliant invoices and collect UPI or bank transfer payments from Indian customers — with zero per-transaction fees.

The REST API accepts JSON and returns JSON. All amounts are in INR. All timestamps are ISO 8601 UTC.

Base URLhttps://yourdomain.com
Versionv0.3
ProtocolHTTPS only
Content-Typeapplication/json
Interactive docs/docs — Swagger UI

Quick Start

From zero to a paid invoice in 5 steps.

1 — Create an account

curl -X POST https://yourdomain.com/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"mypassword","name":"Ravi Kumar"}'

2 — Create an API key

curl -X POST https://yourdomain.com/keys \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{"name":"My App"}'
# Save the raw_key — it won't be shown again: inv_a1b2c3d4e5f6...

3 — Add a UPI payment method

curl -X POST https://yourdomain.com/payment-methods/upi \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{"label":"Business UPI","upi_id":"ravikumar@okaxis","upi_name":"Ravi Kumar","is_default":true}'

4 — Create and send an invoice

# Create
curl -X POST https://yourdomain.com/invoices \
  -H "Authorization: Bearer inv_a1b2c3d4e5f6..." \
  -H "Content-Type: application/json" \
  -d '{
    "customer_name": "Priya Sharma",
    "customer_email": "priya@example.com",
    "line_items": [
      {"name":"Website Design","quantity":1,"rate":25000},
      {"name":"Logo Design","quantity":1,"rate":8000}
    ],
    "gst_rate": 18,
    "due_date": "2025-05-30"
  }'

# Send (emails customer with PDF + payment link)
curl -X POST https://yourdomain.com/invoices/<invoice_id>/send \
  -H "Authorization: Bearer inv_a1b2c3d4e5f6..."

5 — Confirm payment (after customer pays)

curl -X POST https://yourdomain.com/invoices/<invoice_id>/confirm-payment \
  -H "Authorization: Bearer inv_a1b2c3d4e5f6..."

Python helper

import httpx

API_KEY = "inv_a1b2c3d4e5f6..."
BASE    = "https://yourdomain.com"

def api(method, path, **kwargs):
    h = kwargs.pop("headers", {})
    h["Authorization"] = f"Bearer {API_KEY}"
    r = httpx.request(method, BASE + path, headers=h, **kwargs)
    r.raise_for_status()
    return r.json()

# Create invoice
invoice = api("POST", "/invoices", json={
    "customer_name": "Priya Sharma",
    "customer_email": "priya@example.com",
    "line_items": [{"name": "Consulting", "quantity": 10, "rate": 2000}],
    "gst_rate": 18,
})
api("POST", f"/invoices/{invoice['id']}/send")

Authentication

Pass a token in every request to a protected endpoint:

Authorization: Bearer <token>

JWT (session token)

Returned by /auth/login and /auth/signup. Valid for 7 days. Best for browser sessions and one-off scripts.

API Key

Created via POST /keys. Format: inv_ prefix + 48 hex chars. The raw key is shown only once at creation — copy it immediately. API keys are rate-limited per plan and are the recommended choice for backend integrations.

Never expose API keys in client-side JavaScript or public repositories. Rotate compromised keys via the dashboard → API Keys page.

Rate Limits

Rate limits apply to API key requests only. JWT (browser) requests are not rate-limited.

Exceeding a limit returns 429 Too Many Requests.

Errors

All errors return a JSON body:

{ "detail": "Human-readable message" }
StatusMeaning
400Bad request — invalid input
401Missing or invalid Authorization token
403Authenticated but not permitted (e.g., another user's invoice)
404Resource not found
409Conflict — duplicate UTR, invoice already paid, etc.
422Validation failure — check field errors in response body
429Rate limit exceeded
500Internal server error

Invoice & Payment Flow

draft ──► sent ──► pending ──► paid
                       │
                   cancelled
StatusWhat it means
draftCreated, not shared. Can be edited or deleted.
sentMerchant called /send. Customer received the payment link.
pendingCustomer submitted a UTR — awaiting merchant confirmation.
paidMerchant confirmed receipt. Final state.
cancelledMerchant cancelled the invoice.

Customer experience

  1. Customer opens the link /pay/{invoice_id}
  2. Page shows a UPI QR code and / or bank transfer details
  3. Customer pays via GPay / PhonePe / NEFT, notes the UTR
  4. Customer pastes the UTR into the form and clicks Submit
  5. Merchant verifies in their banking app → clicks Confirm in dashboard
  6. Customer receives a payment confirmation email (if email was provided)

Register a URL to receive real-time POST notifications on payment events.

Register an endpoint

  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://myapp.com/hooks/invoice"}'

# Response contains webhook_secret — save it, shown only once

Payload

{
  "event":          "payment.confirmed",
  "invoice_id":     "3fa85f64-...",
  "invoice_number": "INV-2025-0042",
  "amount":         38940.00,
  "currency":       "INR",
  "customer_name":  "Priya Sharma",
  "utr":            "421234567890"
}

Verifying signatures

Every delivery sends an X-Invoice-Signature header — HMAC-SHA256 of the raw body signed with your webhook secret.

import hmac, hashlib

def verify(raw_body: bytes, sig: str, secret: str) -> bool:
    expected = hmac.new(secret.encode(), raw_body, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, sig)
Always verify the signature before processing. Reject requests where verification fails.

Events

EventFired when
payment.confirmedMerchant calls POST /invoices/{id}/confirm-payment
payment.rejectedMerchant calls POST /invoices/{id}/reject-payment
Webhooks are sent once with no automatic retry. Check GET /webhooks/deliveries for failed attempts.

GET /invoices JWT API Key

List invoices with optional filtering and pagination. Results are cached for 30 seconds.

Query parameters

ParamTypeDefaultDescription
statusstring—draft | sent | pending | paid | cancelled

Response — 200 — Array<InvoiceOut>

GET /invoices/{invoice_id} JWT API Key

Fetch a single invoice by UUID.

PATCH /invoices/{invoice_id} JWT API Key

Update an invoice. Only draft invoices can be fully edited. All fields are optional.

GET /invoices/{invoice_id}/pdf JWT API Key

Download the GST-compliant A4 PDF. Returns application/pdf with Content-Disposition: attachment. PDF includes CGST + SGST split, amount in words, and a UPI QR code.

POST /invoices/{invoice_id}/send JWT API Key

Mark invoice as sent, generate a payment link, and email the customer with a PDF attachment. Email is non-blocking — the API returns immediately.

Set customer_email on the invoice for email delivery. The payment link is always generated regardless.

Response — 200

{
  "invoice_id":     "3fa85f64-...",
  "invoice_number": "INV-2025-0042",
  "payment_url":    "https://yourdomain.com/pay/3fa85f64-...",
  "amount":         38940.00,
  "currency":       "INR",
  "status":         "sent"
}
DEL /invoices/{invoice_id} JWT API Key

Cancel an invoice. Cannot cancel paid invoices. Returns 204 No Content.

Payment Endpoints

POST /invoices/{invoice_id}/confirm-payment JWT API Key

Confirm that payment was received. Sets invoice → paid, payment → confirmed. Sends a receipt email to the customer (non-blocking) and fires the payment.confirmed webhook.

POST /invoices/{invoice_id}/reject-payment JWT API Key

Reject a submitted UTR. Reverts invoice to sent and fires the payment.rejected webhook.

Query parameters

ParamDefault
reason"Payment could not be verified"
GET /invoices/{invoice_id}/payments JWT API Key

List all payment attempts for an invoice, including rejected ones.
